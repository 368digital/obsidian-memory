# Система баз (Bases) для obsidian-memory

## Проблема

Сессии изолированы друг от друга. Нет способа понять:
- Какие сессии относятся к одной теме
- Какое текущее состояние направления работы
- Как сессии связаны между собой

## Решение: Базы

База — контекстный блок (домен работы), к которому привязываются сессии. Claude определяет принадлежность по описанию базы. obsidian-memory остаётся надстройкой — не управляет проектом, только следит за контекстом.

## Формат файла базы

Расположение: `claude-memory/bases/{Название}.md`

```markdown
---
name: Плагин Obsidian
description: Obsidian-плагин для визуализации и синхронизации claude-memory
paths:
  - obsidian-claude-memory/src/
  - obsidian-claude-memory/manifest.json
tags: [obsidian, plugin, typescript]
created: 2026-04-13
---

## Описание
Плагин Obsidian для работы с claude-memory: sidebar, dashboard, timeline,
синхронизация через ChangedWriter, управление сессиями через SessionGuardian.

## Ключевые компоненты
- ChangedWriter — polling внешних изменений через vault.adapter
- SessionGuardian — закрытие stale сессий, создание stubs
- DashboardView, TimelineView, SidebarView — UI компоненты

## Хронология
- 2026-04-13 — v0.2.0 опубликован. ChangedWriter на vault events. [[2026-04-13 — Инициализация obsidian-memory]]
- 2026-04-13 — ChangedWriter переписан на adapter polling. [[2026-04-13 15-30 — Починка сессий и ChangedWriter]]
```

### Поля frontmatter

- `name` — человекочитаемое название
- `description` — краткое описание для Claude (определяет принадлежность сессий)
- `paths` — подсказка, какие пути файлов относятся к базе (ориентир, не жёсткое правило)
- `tags` — для группировки и фильтрации
- `created` — дата создания

### Хронология

Контрольные точки состояния базы. Каждая запись: **дата — что готово/сломано/изменилось** + wikilink на сессию. Обновляется при завершении сессии, если были значимые изменения в базе. Не каждый коммит — только то, что меняет состояние.

### Новые поля

- `bases` — массив wikilinks на базы (многие-ко-многим)
- `continues` — wikilink на предыдущую сессию в цепочке (опционально)
- `## Связи` — дублирует wikilinks в теле для графа Obsidian
- Файлы — обычный текст с путями (без wikilinks, граф чистый)

## Флоу при старте диалога

1. Хук SessionStart → напоминает создать сессию
2. Claude читает `.changed` (если есть)
3. Claude читает `MEMORY.md`
4. Claude читает `claude-memory/bases/*.md` (список баз + описания)
5. Claude спрашивает: "К какой базе относится? 1) Плагин Obsidian 2) Скилл 3) Новая база"
6. Пользователь выбирает (можно несколько)
7. Claude читает хронологию выбранных баз — понимает текущее состояние
8. Claude создаёт файл сессии с `bases` и `continues`
9. Работа начинается

### Определение `continues`

Claude смотрит последнюю сессию в выбранной базе и предлагает как продолжение. Если новая тема в рамках базы — `continues` пустое.

### Переключение базы в процессе

Пользователь может переключить/добавить базу командой `/obs-mem base Название` или естественным языком. Claude добавляет базу в `bases` сессии и читает её хронологию.

## Команды `/obs-mem`

```
/obs-mem init              — инициализация claude-memory/ в проекте
/obs-mem base              — показать текущие базы сессии + список всех
/obs-mem base Название     — добавить базу к текущей сессии
/obs-mem base new          — создать новую базу (спросит название + описание)
/obs-mem base remove Назв  — убрать базу из сессии
```

## Визуализация в Obsidian

### Sidebar
Список баз: название, количество сессий, последняя активность. Клик → открывает файл базы.

### Dashboard
Карточки баз: описание, хронология последних 3-5 записей, связанные сессии. Фильтр по тегам.

### Timeline
Dropdown/фильтр по базе. По умолчанию все сессии, выбрал базу → только её сессии.

### Парсер
Плагин читает frontmatter баз (name, paths, tags, created) + секции для отображения.
